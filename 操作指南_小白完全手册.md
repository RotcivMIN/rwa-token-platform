## 1. 项目是什么

这是一个 **RWA（真实世界资产）ETF 代币化平台**，把黄金 ETF 等传统金融资产变成区块链上的数字代币。

### 核心功能
| 功能 | 说明 |
|------|------|
| 🪙 **ERC-20 代币** | RWA ETF Gold (RWAG), 初始 1,000,000 枚 |
| ✅ **白名单 (Whitelist)** | 只有 KYC 通过的地址才能持有/转账代币 |
| 🔨 **铸造/销毁 (Mint/Burn)** | 管理员可以增发或销毁代币（对应 ETF 份额增减） |
| 🚫 **交易拦截 (TransactionBlocked)** | 非白名单转账会被拦截并记录 |
| ⏸️ **暂停 (Pause)** | 紧急情况下管理员可冻结所有转账 |
| 📊 **管理面板 (Dashboard)** | 网页端查看和管理所有功能 |

### 技术栈
```
前端:  Next.js + React + Tailwind + DaisyUI + Wagmi + Viem + RainbowKit
合约:  Solidity 0.8.30 + OpenZeppelin v5 + Hardhat
网络:  Ethereum Sepolia 测试网
RPC:   Infura (你的 API Key)
```

## 5. 功能操作指南（E2E 测试）

启动前端后 (`yarn start`)，按以下顺序测试每个功能：

### 5.1 连接钱包

1. 打开 http://localhost:3000
2. 点击右上角 **"Connect Wallet"** 按钮
3. 选择 **MetaMask**
4. 在 MetaMask 弹窗中确认连接
5. 确保网络是 **Sepolia**（如果 MetaMask 提示切换网络，同意即可）

### 5.2 Dashboard（首页）

路径: http://localhost:3000

你应该能看到：
- 代币名称: **RWA ETF Gold**
- 代币符号: **RWAG**
- 总供应量: **1,000,000 RWAG**
- 你的余额: **1,000,000 RWAG**（管理员拥有全部初始代币）
- 白名单状态: **✅ Whitelisted**
- 合约暂停状态: **Active (Not Paused)**

### 5.3 Whitelist Manager（白名单管理）

路径: http://localhost:3000/whitelist

**测试 1: 查看白名单**
- 你的地址应该已在白名单中（部署时自动添加）

**测试 2: 添加新地址到白名单**
1. 在 "Add Address" 输入框输入一个测试地址，例如:
   ```
   0x0000000000000000000000000000000000000001
   ```
   > 或者用你 MetaMask 的另一个账户地址
2. 点击 **"Add to Whitelist"**
3. MetaMask 弹出交易确认 → 点击 **Confirm**
4. 等待交易完成（约 15-30 秒）
5. 刷新页面，白名单计数应该 +1

**测试 3: 移除白名单地址**
1. 找到刚才添加的地址
2. 点击 **"Remove"**
3. MetaMask 确认 → 完成

### 5.4 Mint/Burn Panel（铸造/销毁面板）

路径: http://localhost:3000/mint-burn

**测试 4: 铸造代币 (Mint)**
1. 在 Mint 区域:
   - **To Address**: 输入你的管理员地址 `0xf4B9B94F3a8Eeeb0957b221f5249F50d9ef89420`
   - **Amount**: 输入 `1000`
   - **Reason**: 输入 `Test mint for E2E verification`
2. 点击 **"Mint"**
3. MetaMask 确认
4. 回 Dashboard 查看总供应量变为 **1,001,000**

**测试 5: 销毁代币 (Burn)**
1. 在 Burn 区域:
   - **From Address**: 你的管理员地址
   - **Amount**: 输入 `500`
   - **Reason**: 输入 `Test burn - ETF redemption`
2. 点击 **"Burn"**
3. MetaMask 确认
4. 总供应量变为 **1,000,500**

**测试 6: 暂停/恢复 (Pause/Unpause)**
1. 点击 **"Pause Contract"**
2. MetaMask 确认
3. 合约状态变为 **Paused** — 此时所有转账/铸造/销毁都会失败
4. 点击 **"Unpause Contract"** 恢复正常

### 5.5 Transactions（交易日志）

路径: http://localhost:3000/transactions

- 你之前的所有操作（Mint、Burn、白名单变更）都会显示在这里
- 可以按 **Tab 标签** 筛选不同类型的事件
- 每条记录包含: 交易哈希、区块号、时间、参数

### 5.6 Transfer 测试（可选进阶）

3. 测试白名单内转账成功 / 白名单外转账失败

## 6. 常见问题排查

### ❌ "Couldn't find a package.json"
```
原因: 没有在项目目录下执行命令
解决: cd "E:\5208 token Tech\rwa-token-platform" 然后再执行
```

### ❌ MetaMask 显示 "Wrong Network"
```
解决: MetaMask 左上角切换到 Sepolia 测试网
```

### ❌ 交易失败 "insufficient funds for gas"
```
原因: Sepolia ETH 不够支付 Gas
解决: 去水龙头领取免费测试 ETH（见 4.4 节）
```

### ❌ 页面显示 "Contract not found" 或数据全为 0
```
可能原因:
1. scaffold.config.ts 没有指向 Sepolia → 检查 targetNetworks
2. .env.local 的 INFURA_API_KEY 为空 → 检查 packages/nextjs/.env.local
3. 浏览器缓存 → 硬刷新 Ctrl+Shift+R
```

### ❌ "yarn: command not found"
```
解决: 确保安装了 Node.js (https://nodejs.org/) 和 Yarn
安装 Yarn: npm install -g yarn
```

### ❌ 页面打不开 / 端口被占用
```
解决: 
1. 确认 yarn start 没有报错
2. 如果端口 3000 被占用，yarn start 会自动用 3001
3. 看终端输出的 URL 是什么
```

### ❌ Mint/Burn 交易被 Revert
```
可能原因:
1. 不是 Owner 地址在操作 → 确认 MetaMask 连接的是管理员账户
2. 合约被暂停了 → 先 Unpause
3. 目标地址不在白名单 → 先添加白名单
```
